# Lombok and Common Utility Libraries

The most headache-inducing part of writing Java is **boilerplate code** — a single POJO needs getter/setter/toString/equals/hashCode, adding up to dozens of lines; every class has to declare a logger manually; constructors pile up in overloads. You can't skip this code, but writing it is tedious. **Lombok** was born to eliminate boilerplate — one annotation replaces dozens of lines.

But Lombok is just the tip of the utility-library iceberg. The Java ecosystem also has Apache Commons, Guava, Hutool, Vavr, and more, covering strings, collections, caching, IO, functional programming, and everything in between. This chapter walks through all of them at once.

## 1. Lombok

**Lombok** is the "code-generation magic" open-sourced in 2009 — it uses an **annotation processor** to inject methods into bytecode at compile time. You only write annotations, and the compiled `.class` file contains the full getter/setter.

### 1.1 Installation

```xml
<dependency>
    <groupId>org.projectlombok</groupId>
    <artifactId>lombok</artifactId>
    <version>1.18.30</version>
    <scope>provided</scope>
</dependency>
```

IDEA needs the Lombok plugin (built into newer versions), and so does Eclipse. The annotation processor does its work at compile time; if the IDE doesn't recognize it, it'll mark everything red.

### 1.2 @Data: One Annotation Covers All

```java
import lombok.*;

@Data                   // Equivalent to @Getter + @Setter + @ToString + @EqualsAndHashCode + @RequiredArgsConstructor
public class User {
    private Long id;
    private String name;
    private Integer age;
}
```

A single `@Data` line replaces five — it auto-generates:

- getter/setter for all fields
- `toString()`
- `equals()` / `hashCode()`
- `@RequiredArgsConstructor` (constructor for final fields)

### 1.3 Common Annotations

| Annotation | What it generates |
| --- | --- |
| `@Getter` / `@Setter` | getter/setter |
| `@ToString` | `toString` |
| `@EqualsAndHashCode` | `equals` / `hashCode` |
| `@NoArgsConstructor` | No-arg constructor |
| `@AllArgsConstructor` | All-args constructor |
| `@RequiredArgsConstructor` | Constructor for final/@NonNull fields (recommended for dependency injection) |
| `@Data` | Combination of the above five |
| `@Builder` | Builder pattern |
| `@Slf4j` | `private static final Logger log = ...` |
| `@Log` | Uses `java.util.logging` |
| `@SneakyThrows` | Sneaky-throws checked exceptions (don't overuse) |
| `@Value` | Immutable class (all final fields) |
| `@NonNull` | Auto null check |
| `@Cleanup` | Auto close (shorthand for try-with-resources) |

### 1.4 @Builder: Fluent Construction

```java
@Builder
@Data
public class User {
    private Long id;
    private String name;
    private Integer age;
}

// Usage
User u = User.builder()
    .id(1L)
    .name("Alice")
    .age(20)
    .build();
```

The Builder pattern's advantage — optional parameters combine freely, and many parameters stay readable. Lombok's `@Builder` auto-generates the Builder inner class, saving you from writing dozens of lines of template by hand.

### 1.5 @Slf4j: Auto-Declare the Logger

```java
@Slf4j
public class UserService {
    public void login(String name) {
        log.info("User {} logged in", name);   // Use log directly
    }
}
```

No need to write `private static final Logger log = LoggerFactory.getLogger(...)` in every class — one `@Slf4j` handles it.

### 1.6 @RequiredArgsConstructor: Constructor Injection

Spring recommends **constructor injection** (instead of `@Autowired` field injection), and Lombok makes it one line:

```java
@Service
@RequiredArgsConstructor
public class UserService {
    private final UserRepository userRepo;     // final -> goes into the constructor
    private final EmailService emailService;
    private final CacheService cacheService;
    // Lombok auto-generates a 3-arg constructor; Spring auto-injects
}
```

`final` fields are included in the constructor by `@RequiredArgsConstructor` — Spring sees the constructor and injects automatically.

### 1.7 @SneakyThrows and @Cleanup

```java
// @SneakyThrows: no try-catch and no throws needed
@SneakyThrows
public String read(String path) {
    return Files.readString(Path.of(path));   // No need to declare IOException
}

// @Cleanup: auto close
public void copy(String src, String dst) throws IOException {
    @Cleanup InputStream in = new FileInputStream(src);
    @Cleanup OutputStream out = new FileOutputStream(dst);
    in.transferTo(out);
    // Auto in.close() / out.close()
}
```

These two are rather "magical" — `@SneakyThrows` uses bytecode tricks to sneakily throw checked exceptions (bypassing compiler checks); overuse hurts readability. `@Cleanup` isn't as standardized as `try-with-resources`; use with caution.

## 2. Apache Commons

Apache Commons is a collection of utility libraries from the Apache Software Foundation, split into multiple modules. The most useful are **Lang** and **Collections**.

### 2.1 Commons Lang

```xml
<dependency>
    <groupId>org.apache.commons</groupId>
    <artifactId>commons-lang3</artifactId>
    <version>3.13.0</version>
</dependency>
```

```java
import org.apache.commons.lang3.StringUtils;
import org.apache.commons.lang3.ObjectUtils;
import org.apache.commons.lang3.Validate;

// Strings
StringUtils.isBlank("  ");              // true (whitespace)
StringUtils.isEmpty("");                // true
StringUtils.isBlank(null);              // true (null-safe)
StringUtils.split("a,b,,c", ",");       // ["a", "b", "c"] (skips empties)
StringUtils.repeat("-", 5);             // "-----"
StringUtils.leftPad("1", 3, '0');       // "001"
StringUtils.substringBetween("a[b]c", "[", "]");  // "b"

// Object utilities
ObjectUtils.defaultIfNull(null, "default");   // "default"
ObjectUtils.firstNonNull(null, null, "a"); // "a"

// Validation
Validate.notNull(arg, "arg must not be null");
Validate.isTrue(i > 0, "i must be positive");
```

The difference between `StringUtils.isBlank` and `String.isEmpty` — `isBlank` treats pure whitespace as blank (`"  "` is blank), while `isEmpty` doesn't. Front-ends often send a bunch of spaces, so `isBlank` is safer.

### 2.2 Commons Collections

```java
import org.apache.commons.collections4.CollectionUtils;
import org.apache.commons.collections4.ListUtils;

// Collection operations
CollectionUtils.isEmpty(list);        // null-safe emptiness check
CollectionUtils.isNotEmpty(list);
ListUtils.union(list1, list2);        // Union
ListUtils.intersection(list1, list2); // Intersection
ListUtils.subtract(list1, list2);     // Difference
```

## 3. Guava: Google's Utility Library

**Guava** is Google's open-source Java utility library — more modern than Commons, with a more elegant API.

```xml
<dependency>
    <groupId>com.google.guava</groupId>
    <artifactId>guava</artifactId>
    <version>32.1.3-jre</version>
</dependency>
```

### 3.1 Collection Enhancements

```java
import com.google.common.collect.*;

// Immutable collections (highly recommended!)
ImmutableList<String> colors = ImmutableList.of("red", "green", "blue");
ImmutableMap<String, Integer> prices = ImmutableMap.of(
    "apple", 5, "banana", 3, "orange", 4);

// Multimap (one key, multiple values)
ListMultimap<String, String> multimap = ArrayListMultimap.create();
multimap.put("fruit", "apple");
multimap.put("fruit", "banana");
multimap.get("fruit");   // [apple, banana]

// BiMap (reverse-lookup value -> key)
BiMap<String, Integer> biMap = HashBiMap.create();
biMap.put("one", 1);
biMap.inverse().get(1);   // "one"

// Table (two-dimensional table)
Table<String, String, Integer> table = HashBasedTable.create();
table.put("Beijing", "population", 21000000);
table.row("Beijing");   // {population=21000000}

// Lists/Sets factories
List<String> list = Lists.newArrayList("a", "b", "c");
Set<String> set = Sets.newHashSet("a", "b");

// Set operations
Sets.union(set1, set2);          // Union
Sets.intersection(set1, set2);   // Intersection
Sets.difference(set1, set2);     // Difference
```

Immutable collections are the essence of Guava — `ImmutableList`/`ImmutableMap` can't be modified after creation, are thread-safe, prevent accidental mutation, and save memory. They're more thorough than `Collections.unmodifiableList` (the latter is only a view; if the underlying collection changes, so does the view).

### 3.2 Cache: Local Cache

```java
import com.google.common.cache.*;

Cache<String, User> cache = CacheBuilder.newBuilder()
    .maximumSize(1000)                          // At most 1000 entries
    .expireAfterWrite(10, TimeUnit.MINUTES)     // Expire 10 minutes after write
    .expireAfterAccess(5, TimeUnit.MINUTES)     // Expire 5 minutes after access
    .recordStats()                              // Record statistics
    .build();

// Write
cache.put("user:1", new User(1, "Alice"));

// Read (auto-load)
User u = cache.get("user:1", () -> loadFromDb(1));   // If absent, call lambda to load

// Stats
CacheStats stats = cache.stats();
stats.hitRate();        // Hit rate
stats.evictionCount();  // Eviction count
```

`LoadingCache` goes one step further — specify a `CacheLoader` at build time to auto-load on demand:

```java
LoadingCache<String, User> loadingCache = CacheBuilder.newBuilder()
    .maximumSize(1000)
    .build(new CacheLoader<String, User>() {
        @Override
        public User load(String key) {
            return loadFromDb(key);   // Auto-called on cache miss
        }
    });

User u = loadingCache.get("user:1");   // Auto-loads
```

### 3.3 EventBus: In-Process Events

```java
import com.google.common.eventbus.*;

EventBus bus = new EventBus();

// Subscribe
class OrderEventHandler {
    @Subscribe
    public void onOrder(OrderEvent event) {
        System.out.println("Handle order: " + event);
    }
}
bus.register(new OrderEventHandler());

// Publish
bus.post(new OrderEvent("ORD-001", 99.5));
// Auto-calls onOrder
```

EventBus implements "publish-subscribe" — publishers don't need to know who subscribes; loosely coupled.

## 4. Hutool: A Chinese-Origin Utility Toolkit

**Hutool** is a utility toolkit contributed by Chinese developers — its API uses Chinese-style naming, is feature-complete, and feels friendly to use.

```xml
<dependency>
    <groupId>cn.hutool</groupId>
    <artifactId>hutool-all</artifactId>
    <version>5.8.24</version>
</dependency>
```

```java
import cn.hutool.core.util.StrUtil;
import cn.hutool.core.collection.CollUtil;
import cn.hutool.core.date.DateUtil;
import cn.hutool.core.io.FileUtil;
import cn.hutool.crypto.SecureUtil;
import cn.hutool.http.HttpUtil;
import cn.hutool.json.JSONUtil;

// Strings
StrUtil.isBlank("  ");                 // true
StrUtil.format("User {}'s age {}", "Alice", 20);  // Parameterized

// Collections
CollUtil.newArrayList("a", "b");
CollUtil.isEmpty(list);

// Files

FileUtil.writeUtf8String("content", "out.txt");

HttpUtil.post(url, jsonBody);

SecureUtil.sha1("password");
```

Hutool's strength is "out of the box" — HTTP, JSON, crypto, Excel, regex, caching all included; it's very popular in Chinese projects.

## 5. Vavr: Functional Utilities

**Vavr** (formerly Javaslang) brings Scala-style functional utilities to Java — `Option`/`Try`/`Tuple`/`Stream`/functional collections.

```xml
<dependency>
    <groupId>io.vavr</groupId>
    <artifactId>vavr</artifactId>
    <version>0.10.4</version>
</dependency>
```

```java
import io.vavr.control.Try;
import io.vavr.control.Option;
import io.vavr.collection.List;
import io.vavr.Tuple;

// Try: replaces try-catch, handles exceptions functionally
Try<Integer> result = Try.of(() -> 1 / 0);
result.isFailure();           // true
result.getOrElse(-1);         // -1
result.recover(e -> -1);      // On failure, return -1

// Option: replaces null, more functional than java.util.Optional
Option<String> name = Option.of(null);
name.getOrElse("anonymous");   // "anonymous"

// Functional collections (immutable, more powerful than Stream)
List.range(0, 5);             // List(0,1,2,3,4)
List.of(1, 2, 3)
    .map(x -> x * 2)
    .filter(x -> x > 2)
    .sum();                   // 10

// Tuple (Java has no tuples; Vavr provides them)
var t = Tuple.of("Alice", 20, true);
t._1;   // "Alice"
t._2;   // 20
```

`Try` is the essence of Vavr — treats exceptions as values, chains recover/transform, and avoids try-catch everywhere. But Vavr's style is aggressive; if the team isn't comfortable with functional programming, use with caution.

class MiniCache<K, V> {
    private final ConcurrentHashMap<K, V> data = new ConcurrentHashMap<>();
    private final ConcurrentHashMap<K, Long> expireAt = new ConcurrentHashMap<>();
    private final long ttlMs;
    private final Function<K, V> loader;
    private final AtomicInteger hits = new AtomicInteger();
    private final AtomicInteger misses = new AtomicInteger();

    public MiniCache(long ttlMs, Function<K, V> loader) {
        this.ttlMs = ttlMs; this.loader = loader;
    }

    public V get(K key) {
        Long exp = expireAt.get(key);
        if (exp != null && System.currentTimeMillis() < exp) {
            V v = data.get(key);
            if (v != null) { hits.incrementAndGet(); return v; }
        }
        // 加载
        misses.incrementAndGet();
        V v = loader.apply(key);
        data.put(key, v);
        expireAt.put(key, System.currentTimeMillis() + ttlMs);
        return v;
    }

    public int hits() { return hits.get(); }
    public int misses() { return misses.get(); }
    public double hitRate() {
        int total = hits.get() + misses.get();
        return total == 0 ? 0 : (double) hits.get() / total;
    }
}

// 5. EventBus (模拟 Guava EventBus)
class EventBus {
    private final Map<Class<?>, List<Consumer<?>>> subscribers = new ConcurrentHashMap<>();
    public <T> void subscribe(Class<T> eventType, Consumer<T> handler) {
        subscribers.computeIfAbsent(eventType, k -> new CopyOnWriteArrayList<>()).add(handler);
    }
    @SuppressWarnings("unchecked")
    public void publish(Object event) {
        List<Consumer<?>> handlers = subscribers.get(event.getClass());
        if (handlers != null) {
            for (Consumer<?> h : handlers) ((Consumer<Object>) h).accept(event);
        }
    }
}

                i++;

                sb.append(tpl.charAt(i));
            }
        }
        return sb.toString();
    }

> **Key observations**:
> - **The `User` class has 30+ lines of boilerplate written by hand** — Lombok's single `@Data` generates it all, cutting more than half the code.
> - **`ImmutableList.append` returns a new list; the original is unchanged** — the core property of immutability, thread-safe.
> - **`Multimap` maps one key to multiple values** — more convenient than `Map＜K, List＜V＞＞`.
> - **The first cache access is a miss taking 50ms, the next two are hits at nearly 0ms** — the value of caching is visible at a glance.
> - **EventBus publishers don't need to know subscribers** — loosely coupled event-driven design.

## 7. Choosing Utility Libraries

Utility libraries are great, but more isn't always better:

1. **Don't overlap** — one StringUtils is enough (Commons or Hutool); don't pull in three at once.
2. **Mind the size** — Guava is 5MB+, Hutool 2MB+; weigh this for mobile/containerized scenarios.
3. **JDK first** — Java 8+'s `List.of`/`Map.of`/`Optional`/`Stream` have replaced parts of Guava. If the JDK can do it, don't pull in a library.
4. **Maintainability** — pick actively maintained libraries. Commons Lang still updates but is less active than Guava.
5. **Team familiarity** — if nobody on the team knows Vavr, introducing it is a burden.

**Practical advice**:

- Backend project standard kit: Lombok + Commons Lang3 + Guava (or Hutool).
- Spring Boot ships with Logback, Jackson, Hibernate Validator — no need to add them.
- Prefer JDK's `List.of`/`Map.of` for immutable collections; reach for Guava only in complex scenarios.
- For caching, use Guava/Caffeine on a single machine, Redis for distributed.
