# Hands-On: Todo CLI

You've learned so much — collections, IO, concurrency, reflection, Stream, date-time — but knowledge right now is a string of scattered pearls; you need a thread to string them into a necklace. That thread is **a real project**.

In this stage we'll build a **Todo CLI (Command Line Interface)** — typing commands in the terminal to manage your todos, just like `git`. It's not large, but it has everything: command parsing, data persistence, priority sorting, tag filtering, and Stream-based statistics. We'll put every piece of core Java we've learned to work and see how they cooperate.

## 1. Requirements Analysis

The CLI we're building supports these commands:

```
todo add "买菜" -p high -t 生活 -t 周末     # 添加待办 (优先级 + 标签)
todo list [--all] [--tag 生活] [--priority high]  # 列出待办
todo done <id>                              # 标记完成
todo delete <id>                            # 删除
todo edit <id> -t "新内容"                  # 编辑
todo clear --done                           # 清除已完成
todo stats                                  # 统计
```

Feature requirements:
- **Create, Read, Update, Delete** — basic CRUD.
- **Priority** — high / medium / low, which affects ordering.
- **Tags** — a todo can have multiple tags, and you can filter by tag.
- **Persistence** — save to a file so nothing is lost on restart.
- **Statistics** — use Stream to compute completion rate, group by tag, and more.

## 2. Architecture Design

A good project starts with design. We'll use two patterns to keep the code clean:

### 2.1 Command Pattern

Each command is an object with an `execute` method. The main program parses the input and dispatches to the matching command. Adding a new command doesn't require touching the main flow — that's the open-closed principle at work.

```
输入 "add 买菜 -p high"
  ↓
解析: command=add, args=[买菜], options={p: high}
  ↓
分发: CommandRegistry.get("add").execute(args, options)
  ↓
AddCommand 执行: 创建 Todo, 存入 Repository
```

### 2.2 Repository Pattern

Data access is encapsulated in `TodoRepository` — the business logic doesn't care where data lives (memory / file / database); it just calls `repository.save(todo)`. Swap the storage and you only change the Repository; the business layer stays put.

```
Command → Repository → Storage (文件/内存)
```

### 2.3 Overall Class Diagram

```
Main (入口)
 ├ CommandLineParser (解析命令行)
 ├ CommandRegistry (命令注册表)
 │   ├ AddCommand
 │   ├ ListCommand
 │   ├ DoneCommand
 │   ├ DeleteCommand
 │   ├ EditCommand
 │   ├ ClearCommand
 │   └ StatsCommand
 └ TodoRepository (数据访问)
     ├ Todo (实体)
     └ TodoStorage (文件持久化)
```

## 3. Core Entity Design

### 3.1 The Todo Entity

```java
public class Todo {

    private Set<String> tags;
    private boolean done;
    private LocalDateTime createdAt;
```

Using an enum for priority is safer than a string — the compiler catches typos for you. The `weight` field drives sorting.

## 4. Command Implementation Walkthrough

### 4.1 AddCommand

Parse the `title` and the options `-p` (priority) and `-t` (tag, can repeat), build a `Todo` object, and save it to the Repository.

```java
public void execute(CommandLine cli) {
    String title = String.join(" ", cli.args);
    Todo todo = new Todo(title);
    if (cli.options.containsKey("p")) {
        todo.setPriority(Priority.valueOf(cli.options.get("p").get(0).toUpperCase()));
    }
    if (cli.options.containsKey("t")) {
        todo.setTags(new HashSet<>(cli.options.get("t")));
    }
    repository.save(todo);
    System.out.println("已添加: " + todo);
}
```

### 4.2 ListCommand

Fetch all todos and sort by priority then creation time. Supports the filter options `--all` (include completed), `--tag`, and `--priority`.

```java
public void execute(CommandLine cli) {
    Stream<Todo> stream = repository.findAll().stream();
    if (!cli.options.containsKey("all")) {
        stream = stream.filter(t -> !t.isDone());   // 默认只显示未完成
    }
    if (cli.options.containsKey("tag")) {
        String tag = cli.options.get("tag").get(0);
        stream = stream.filter(t -> t.getTags().contains(tag));
    }
    // 按优先级 + 创建时间排序
    List<Todo> list = stream
        .sorted(Comparator.comparingInt((Todo t) -> t.getPriority().weight)
                          .thenComparing(Todo::getCreatedAt))
        .collect(Collectors.toList());
    // 打印表格
    printTable(list);
}
```

The Stream API shines here — filter, sort, and collect, all in one fluent chain.

### 4.3 StatsCommand

Use Stream to compute statistics — totals, completed count, completion rate, group by priority, group by tag.

```java
public void execute(CommandLine cli) {
    List<Todo> all = repository.findAll();
    long total = all.size();
    long done = all.stream().filter(Todo::isDone).count();
    double rate = total == 0 ? 0 : (double) done / total * 100;

    System.out.printf("总数: %d, 完成: %d, 完成率: %.1f%%%n", total, done, rate);

    // 按优先级分组统计
    Map<Priority, Long> byPriority = all.stream()
        .collect(Collectors.groupingBy(Todo::getPriority, Collectors.counting()));
    System.out.println("按优先级: " + byPriority);

    // 按标签分组 (flatMap 展开)
    Map<String, Long> byTag = all.stream()
        .flatMap(t -> t.getTags().stream())
        .collect(Collectors.groupingBy(Function.identity(), Collectors.counting()));
    System.out.println("按标签: " + byTag);
}
```

`flatMap` "flattens" each todo's tag set into a single stream — it's the Stream tool of choice for nested structures.

## 5. Persistence Design

The `TodoRepository` interface isn't bound to any specific storage. We'll implement a `FileTodoRepository` that saves to a file. But the Piston online environment restricts file paths, so we also implement an `InMemoryTodoRepository` to keep things runnable.

```java
interface TodoRepository {
    Todo save(Todo todo);
    Optional<Todo> findById(Long id);
    List<Todo> findAll();
    void deleteById(Long id);
    void update(Todo todo);
}

// 文件实现: 每行一个 Todo, 格式 id|title|priority|done|tags|createdAt
class FileTodoRepository implements TodoRepository {
    private final Path file;
    public FileTodoRepository(Path file) { this.file = file; }

    public List<Todo> findAll() {
        if (!Files.exists(file)) return new ArrayList<>();
        return Files.lines(file)
            .map(this::parseLine)
            .filter(Objects::nonNull)
            .collect(Collectors.toList());
    }
    // save / delete / update 类似, 全量重写文件
}
```

A real project would use a database (SQLite/H2); here a file-based approach is enough and simple.

class CommandLine {
    String command;
    List<String> args = new ArrayList<>();
    Map<String, List<String>> options = new LinkedHashMap<>();

    static CommandLine parse(String input) {
        CommandLine cli = new CommandLine();
        // 简化: 按空格分, 但引号内的内容算一个参数
        List<String> tokens = tokenize(input);
        if (tokens.isEmpty()) return cli;
        cli.command = tokens.get(0);
        for (int i = 1; i < tokens.size(); i++) {
            String t = tokens.get(i);
            if (t.startsWith("--")) {
                String key = t.substring(2);
                cli.options.computeIfAbsent(key, k -> new ArrayList<>());
                if (i + 1 < tokens.size() && !tokens.get(i + 1).startsWith("-")) {
                    cli.options.get(key).add(tokens.get(++i));
                }
            } else if (t.startsWith("-")) {
                String key = t.substring(1);
                cli.options.computeIfAbsent(key, k -> new ArrayList<>());
                if (i + 1 < tokens.size() && !tokens.get(i + 1).startsWith("-")) {
                    cli.options.get(key).add(tokens.get(++i));
                }
            } else {
                cli.args.add(t);
            }
        }
        return cli;
    }

    private static List<String> tokenize(String input) {
        List<String> tokens = new ArrayList<>();
        StringBuilder sb = new StringBuilder();
        boolean inQuote = false;
        for (char c : input.toCharArray()) {
            if (c == '"') { inQuote = !inQuote; continue; }
            if (c == ' ' && !inQuote) {
                if (sb.length() > 0) { tokens.add(sb.toString()); sb.setLength(0); }
            } else { sb.append(c); }
        }
        if (sb.length() > 0) tokens.add(sb.toString());
        return tokens;
    }
}

// ============ 仓储 ============
class TodoRepository {
    private final List<Todo> todos = new ArrayList<>();
    private long nextId = 1;

    public Todo save(String title, Todo.Priority priority, Set<String> tags) {
        Todo t = new Todo(nextId++, title);
        t.setPriority(priority);
        if (tags != null) t.setTags(new LinkedHashSet<>(tags));
        todos.add(t);
        return t;
    }

    public List<Todo> findAll() { return new ArrayList<>(todos); }

    public Optional<Todo> findById(long id) {
        return todos.stream().filter(t -> t.getId() == id).findFirst();
    }

    public boolean deleteById(long id) {
        return todos.removeIf(t -> t.getId() == id);
    }

    public void clearDone() { todos.removeIf(Todo::isDone); }
}

        Todo.Priority p = Todo.Priority.MEDIUM;
        if (cli.options.containsKey("p")) {
            try { p = Todo.Priority.valueOf(cli.options.get("p").get(0).toUpperCase()); }
            catch (Exception e) { System.out.println("无效优先级, 用 high/medium/low"); return; }

        Set<String> tags = new LinkedHashSet<>(cli.options.getOrDefault("t", Collections.emptyList()));
        Todo t = repo.save(title, p, tags);
        System.out.println("✓ 已添加 #" + t.getId() + ": " + t.getTitle());
    }

    private void doList(CommandLine cli) {
        Stream<Todo> stream = repo.findAll().stream();
        boolean showAll = cli.options.containsKey("all");
        if (!showAll) stream = stream.filter(t -> !t.isDone());

            stream = stream.filter(t -> t.getTags().contains(tag));

            String pStr = cli.options.getOrDefault("p", cli.options.get("priority")).get(0);

## 8. Extension Ideas

This CLI can grow in many directions:

1. **File persistence** — implement `FileTodoRepository`, one Todo per line, read with `Files.lines`.
2. **JSON storage** — serialize to JSON with Jackson/Gson for better readability.
3. **Multi-user** — add a `User` entity, with an independent Todo list per user.
4. **Reminders** — add a `dueDate` field and use a `ScheduledExecutorService` to check for overdue items.
5. **Web UI** — swap the Repository for a database, add a Spring Boot Controller layer, and turn it into a web app.
6. **Import/Export** — support exporting to CSV/Markdown.
7. **Undo/Redo** — implement `undo`/`redo` with the command pattern plus a history stack.
8. **Easter eggs** — use ANSI color codes to colorize terminal output (e.g., high-priority in red).
