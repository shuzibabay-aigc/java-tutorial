# Data Structures and Algorithms

The longer you code, the more you realize — business requirements change endlessly, but underneath it all sits **data structures + algorithms**. How do you store a list for the fastest lookup? Why is a piece of code slow? How do you find the top K items among a million records? All of it rests on data structures and algorithms. This chapter walks through the core — complexity, linked lists/stacks/queues, trees, hashing, sorting, and common algorithmic tricks — all in one pass.

## I. Complexity Analysis

**Time Complexity** — how an algorithm's running time grows with input size `n`, expressed with **Big-O Notation**.

| Complexity | Name | Example | At n = 1 million |
| --- | --- | --- | --- |
| `O(1)` | Constant | Array index access | 1 step |
| `O(log n)` | Logarithmic | Binary search | 20 steps |
| `O(n)` | Linear | Traversing an array | 1 million steps |
| `O(n log n)` | Linearithmic | Quicksort / merge sort | 20 million steps |
| `O(n²)` | Quadratic | Bubble / selection sort | 1 trillion steps |
| `O(2ⁿ)` | Exponential | Brute-force subsets | Universe-ending |

**Space Complexity** — how extra memory grows with `n`.

Rules of thumb:

- `O(n²)` becomes unbearably slow past n > 10,000 — you must optimize.
- `O(n log n)` is the lower bound for comparison-based sorting — that's as fast as they get.
- `O(log n)` is practically "constant" — the height of a binary tree or balanced tree is log n.

## II. Linear Structures

### 2.1 Arrays

Arrays — contiguous memory, fixed size, O(1) random access, O(n) insertion/deletion. Java's `ArrayList` is array-backed — it auto-grows capacity (by 1.5×) when full.

### 2.2 Linked Lists

Linked lists — non-contiguous memory, nodes chained together, O(1) insert/delete at the head, O(n) random access.

```java
class Node {
    int val;
    Node next;
    Node(int v) { val = v; }
}

// Reverse a singly linked list (a classic interview question)
Node reverse(Node head) {
    Node prev = null, cur = head;
    while (cur != null) {
        Node next = cur.next;
        cur.next = prev;
        prev = cur;
        cur = next;
    }
    return prev;
}
```

`LinkedList` is a doubly linked list — `add`/`remove` at the head or tail is O(1), random `get(i)` is O(n). This is why **`LinkedList` is almost never used** — `ArrayList` is usually faster even for inserts (it's CPU-cache friendly).

### 2.3 Stacks

A stack — Last-In-First-Out (LIFO). `push`/`pop`/`peek` are all O(1).

```java
Deque<Integer> stack = new ArrayDeque<>();
stack.push(1); stack.push(2);
stack.peek();   // 2
stack.pop();    // 2
```

Applications: bracket matching, expression evaluation, the function call stack, DFS. `ArrayDeque` is faster than the legacy `Stack` class — `Stack` extends `Vector` and synchronizes every method, which is expensive.

### 2.4 Queues

A queue — First-In-First-Out (FIFO). `offer`/`poll`/`peek` are all O(1).

```java
Queue<Integer> queue = new ArrayDeque<>();
queue.offer(1); queue.offer(2);
queue.peek();   // 1
queue.poll();   // 1
```

Applications: BFS, producer-consumer, task scheduling. `ArrayDeque` for a plain queue, `PriorityQueue` for a priority queue (a heap), `LinkedBlockingQueue` for a blocking queue (used by thread pools).

## III. Trees

### 3.1 Binary Trees

Each node has at most two children.

```java
class TreeNode {
    int val;
    TreeNode left, right;
}

// Three traversals
void preOrder(TreeNode n) {   // Pre-order: root-left-right
    if (n == null) return;
    System.out.println(n.val);
    preOrder(n.left);
    preOrder(n.right);
}
void inOrder(TreeNode n) {    // In-order: left-root-right (yields sorted order for a BST)
    if (n == null) return;
    inOrder(n.left);
    System.out.println(n.val);
    inOrder(n.right);
}
```

### 3.2 Binary Search Tree (BST)

Left subtree all ＜ root, right subtree all ＞ root. In-order traversal yields a sorted sequence. Search/insert/delete average O(log n), worst case O(n) (when it degenerates into a linked list).

### 3.3 Balanced Trees: Red-Black Tree / AVL

When a BST degenerates into a linked list, it gets slow — **balanced trees** stay balanced through rotations. AVL is strictly balanced (height difference between left and right ≤ 1); the red-black tree is loosely balanced (the longest path is at most twice the shortest).

- **AVL** — many lookups, few inserts/deletes (strictly balanced, fast lookup).
- **Red-black tree** — many inserts/deletes (loosely balanced, fewer rotations). Java's `TreeMap` and `HashMap` (when a bucket's list exceeds 8, it becomes a red-black tree) both use red-black trees.

### 3.4 B-Tree / B+ Tree

Multi-way balanced search trees — each node has many children and many keys. In a B+ tree, all data lives in the leaves, and the leaves are chained into a linked list.

Why do databases use B+ trees instead of red-black trees? Because **disk I/O reads in blocks** — a B+ tree is short and wide, so reading one node (one page) fetches many keys, and the number of I/O operations is small (tree height is low). MySQL indexes are B+ trees.

### 3.5 Heaps

A heap — a complete binary tree where each parent ≥/≤ its children (max-heap / min-heap). `PriorityQueue` is a min-heap.

```java
PriorityQueue<Integer> minHeap = new PriorityQueue<>();
PriorityQueue<Integer> maxHeap = new PriorityQueue<>(Comparator.reverseOrder());

maxHeap.offer(3); maxHeap.offer(1); maxHeap.offer(2);
maxHeap.peek();   // 3 (the max)
maxHeap.poll();   // 3
```

Applications: Top K problems, merging K sorted lists, task scheduling. Heap insert/delete is O(log n).

## IV. Hash Tables

A hash table — the `key` is run through a hash function to compute an array index; average O(1) lookup.

```java
Map<String, Integer> map = new HashMap<>();
map.put("张三", 20);   // hash("张三") % bucketCount -> bucket index
map.get("张三");       // compute the same index, O(1) retrieval
```

### 4.1 Hash Collisions

Different keys hash to the same index — a collision. Two solutions:

- **Separate Chaining** — each bucket holds a linked list. Java's `HashMap` uses this. When a list exceeds 8, it converts to a red-black tree (defense against hash attacks).
- **Open Addressing** — on collision, probe for the next empty slot (linear probing, quadratic probing, double hashing). `ThreadLocalMap` uses this.

### 4.2 Load Factor and Resizing

`HashMap` defaults to 16 buckets with a load factor of 0.75 — when element count exceeds 16 × 0.75 = 12, it resizes to 32 buckets and rehashes. Resizing is expensive (O(n)), so if you know the size in advance, preallocate.

### 4.3 Consistent Hashing

Used in distributed caching — when nodes are added or removed, only the data in the adjacent segment is affected; nothing is rehashed wholesale. Redis Cluster's hash slots follow a similar idea.

## V. Sorting Algorithms

| Algorithm | Average | Worst | Space | Stable |
| --- | --- | --- | --- | --- |
| Bubble | O(n²) | O(n²) | O(1) | ✅ |
| Selection | O(n²) | O(n²) | O(1) | ❌ |
| Insertion | O(n²) | O(n²) | O(1) | ✅ |
| **Quicksort** | O(n log n) | O(n²) | O(log n) | ❌ |
| Merge sort | O(n log n) | O(n log n) | O(n) | ✅ |
| Heap sort | O(n log n) | O(n log n) | O(1) | ❌ |

**Stable** — equal elements keep their relative order. `Arrays.sort` uses quicksort for primitives (unstable) and TimSort (a merge-sort improvement, stable) for objects.

### 5.1 Quicksort

```java
void quickSort(int[] arr, int low, int high) {
    if (low >= high) return;
    int pivot = partition(arr, low, high);
    quickSort(arr, low, pivot - 1);
    quickSort(arr, pivot + 1, high);
}

int partition(int[] arr, int low, int high) {
    int pivot = arr[high];   // Pick the rightmost element as the pivot
    int i = low - 1;          // i is the boundary of the "less-than-pivot" region
    for (int j = low; j < high; j++) {
        if (arr[j] < pivot) {
            i++;
            swap(arr, i, j);
        }
    }
    swap(arr, i + 1, high);
    return i + 1;
}

void swap(int[] arr, int a, int b) {
    int t = arr[a]; arr[a] = arr[b]; arr[b] = t;
}
```

The idea behind quicksort — pick a pivot, put everything smaller on the left and larger on the right, then recursively sort both sides. Worst case O(n²) (already sorted array + picking an endpoint as pivot); randomizing the pivot or taking the median of three avoids this.

### 5.2 Merge Sort

```java
void mergeSort(int[] arr, int[] temp, int low, int high) {
    if (low >= high) return;
    int mid = (low + high) / 2;
    mergeSort(arr, temp, low, mid);
    mergeSort(arr, temp, mid + 1, high);
    merge(arr, temp, low, mid, high);
}

void merge(int[] arr, int[] temp, int low, int mid, int high) {
    int i = low, j = mid + 1, k = low;
    while (i <= mid && j <= high) {
        if (arr[i] <= arr[j]) temp[k++] = arr[i++];
        else temp[k++] = arr[j++];
    }
    while (i <= mid) temp[k++] = arr[i++];
    while (j <= high) temp[k++] = arr[j++];
    System.arraycopy(temp, low, arr, low, high - low + 1);
}
```

Merge sort — divide-and-conquer down to single elements, then merge pairwise. Stable sort, reliably O(n log n), but needs O(n) extra space.

### 5.3 Heap Sort

Build a max-heap → swap the top to the end → shrink the heap → re-heapify. In-place sort, space O(1).

## VI. Common Algorithmic Tricks

### 6.1 Two Pointers

- **Fast and slow pointers** — find the middle of a linked list, detect cycles.
- **Left and right pointers** — binary search, two-sum (on a sorted array).

```java
// Detect a cycle in a linked list
boolean hasCycle(Node head) {
    Node slow = head, fast = head;
    while (fast != null && fast.next != null) {
        slow = slow.next;
        fast = fast.next.next;
        if (slow == fast) return true;
    }
    return false;
}
```

### 6.2 Sliding Window

Maintain a window and slide its left and right pointers — to find the longest/shortest substring.

```java
// Length of the longest substring without repeating characters
int lengthOfLongestSubstring(String s) {
    Map<Character, Integer> last = new HashMap<>();
    int left = 0, max = 0;
    for (int right = 0; right < s.length(); right++) {
        char c = s.charAt(right);
        if (last.containsKey(c) && last.get(c) >= left) {
            left = last.get(c) + 1;
        }
        last.put(c, right);
        max = Math.max(max, right - left + 1);
    }
    return max;
}
```

### 6.3 Binary Search

```java
int binarySearch(int[] arr, int target) {
    int low = 0, high = arr.length - 1;
    while (low <= high) {
        int mid = low + (high - low) / 2;   // Prevents int overflow
        if (arr[mid] == target) return mid;
        if (arr[mid] < target) low = mid + 1;
        else high = mid - 1;
    }
    return -1;
}
```

The precondition for binary search — the array is sorted. Use `mid = low + (high - low) / 2` instead of `(low + high) / 2` — the latter's `low + high` can overflow int (the famous bug Joshua Bloch fixed in JDK 9).

### 6.4 Dynamic Programming Basics

Dynamic Programming (DP) — break a big problem into subproblems, and remember the answers to subproblems to avoid recomputing them.

**Classic: Fibonacci**.

```java
// Recursion (O(2^n), recomputes a lot)
int fib(int n) {
    if (n <= 1) return n;
    return fib(n - 1) + fib(n - 2);
}

// DP (O(n), memoized)
int fibDp(int n) {
    if (n <= 1) return n;
    int[] dp = new int[n + 1];
    dp[0] = 0; dp[1] = 1;
    for (int i = 2; i <= n; i++) dp[i] = dp[i - 1] + dp[i - 2];
    return dp[n];
}
```

**Classic: Climbing Stairs** — n steps, each time you climb 1 or 2. How many ways to the top?

```java
// State transition: dp[i] = dp[i-1] + dp[i-2]
int climbStairs(int n) {
    if (n <= 2) return n;
    int a = 1, b = 2;
    for (int i = 3; i <= n; i++) {
        int c = a + b;
        a = b; b = c;
    }
    return b;
}
```

DP in three steps — **define the state**, **write the transition equation**, **determine the base cases and computation order**.

### 6.5 BFS / DFS

```java
// Level-order traversal of a binary tree (BFS)
List<List<Integer>> levelOrder(TreeNode root) {
    List<List<Integer>> res = new ArrayList<>();
    if (root == null) return res;
    Queue<TreeNode> q = new ArrayDeque<>();
    q.offer(root);
    while (!q.isEmpty()) {
        int size = q.size();
        List<Integer> level = new ArrayList<>();
        for (int i = 0; i < size; i++) {
            TreeNode n = q.poll();
            level.add(n.val);
            if (n.left != null) q.offer(n.left);
            if (n.right != null) q.offer(n.right);
        }
        res.add(level);
    }
    return res;
}
```

BFS — queue, level order, shortest path (unweighted graph). DFS — stack/recursion, depth-first, connectivity, topological sort.

> **What to watch for**:
> - **Quicksort sorts in place** — partition + recurse, O(n log n) on average.
> - **Binary search halves each step** — O(log n); 1 million items take only 20 steps.
> - **Min-heap for Top K** — keep the heap size fixed at K; when it overflows, evict the smallest; what's left is the Top K.
> - **Fast/slow pointers detect cycles** — if the fast pointer catches the slow one, there's a cycle.
> - **Sliding window maintains non-repeating characters** — left and right pointers pinch out a duplicate-free range.
> - **DP turns recursive O(2ⁿ) into O(n)** — memoization avoids recomputation.

## VIII. Chapter Summary

| Concept | Key Point |
| --- | --- |
| Big-O | Growth trend of time/space complexity |
| Linked list | O(1) insert/delete at ends, O(n) random access |
| Stack / Queue | LIFO / FIFO, O(1) operations |
| Binary search tree | In-order is sorted, average O(log n) |
| Red-black tree | Loosely balanced, used by Java `TreeMap`/`HashMap` |
| B+ tree | Database index, short and wide for disk I/O |
| Heap | `PriorityQueue`, Top K problems |
| Hash collision | Separate chaining / open addressing |
| Quicksort | O(n log n) average, O(n²) worst, unstable |
| Merge sort | Stable O(n log n), O(n) space |
| Dynamic programming | State + transition equation + base case |

**Memory hooks**:

- **Big-O shows the trend** — O(1) is best, O(n²) needs optimization.
- **Arrays for fast random access, linked lists for fast inserts/deletes** — choose the trade-off.
- **BST in-order is sorted** — traverse left-root-right.
- **HashMap converts a bucket's list to a red-black tree past 8** — defends against hash attacks.
- **Quicksort averages O(n log n)** — partition + recursion.
- **Binary search is O(log n)** — prerequisite: sorted.
- **DP in three steps** — state, transition, base case.
